# 검증 계획 (Validation Plan)

## 1. 핵심 가설

### 가설 1: 시간 절감
> "PM/기획자가 경쟁사 역기획서를 수동으로 작성할 때 평균 2~3시간이 소요되는 반면, 이 도구를 사용하면 5분 이내에 초안을 얻을 수 있어 **작업 시간을 70% 이상 단축**할 수 있다."

- **검증 방법**: 동일한 사이트(예: notion.so)에 대해 수동 작성 시간 vs. 도구 사용 시간 비교 측정
- **기준**: 수동 작성 2시간 → 도구 사용 후 편집 30분 이내

### 가설 2: 품질 충족도
> "AI가 생성한 역기획서가 PM이 직접 작성한 수준의 **6개 핵심 섹션을 80% 이상 충족**한다."

- **검증 방법**: notion.so, linear.app, toss.im 3개 사이트 생성 결과를 PM 직접 평가
- **평가 기준**: 각 섹션(서비스 개요, UI/UX, 기능명세, 비즈니스 로직, 차별점, 기술 스택)의 완성도를 5점 척도로 평가

### 가설 3: 실무 활용 가능성
> "생성된 역기획서를 별도 편집 없이 또는 최소한의 수정(30분 이내)으로 **실무 보고서로 활용할 수 있다**."

- **검증 방법**: 사용자 인터뷰 (PM 3인) 또는 해커톤 현장 피드백 수집
- **기준**: 사용자 만족도 5점 척도 중 4점 이상

## 2. 성공 지표 (Success Metrics)

### 기술적 지표
| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 역기획서 생성 성공률 | **공개 사이트 기준 80% 이상** | notion.so, linear.app, toss.im, coupang.com 등 10개 사이트 테스트 |
| 평균 생성 소요 시간 | **60초 이내** | `/api/analyze` 응답 시간 측정 (5회 평균) |
| 6개 섹션 완성률 | **100%** | 생성된 마크다운에 h2 섹션 6개 모두 포함 여부 확인 |
| 스크린샷 제공률 | **70% 이상** | PageSpeed API 성공 건수 / 전체 요청 건수 |

### 사용자 경험 지표
| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 사용자 만족도 | **5점 척도 중 4점 이상** | 해커톤 심사위원 및 현장 참여자 피드백 |
| 작업 시간 단축률 | **70% 이상** | 수동 작성 대비 소요 시간 비교 |
| 재사용 의향 | **응답자 70% 이상 "다시 사용하겠다"** | 현장 피드백 설문 |

## 3. 검증 시나리오

### 시나리오 A: 기본 기능 검증 (해커톤 데모)
```
1. notion.so 역기획서 생성 → 60초 이내 완료 확인
2. linear.app 역기획서 생성 → 6개 섹션 포함 확인
3. toss.im 역기획서 생성 → 스크린샷 + 와이어프레임 표시 확인
4. 마크다운 다운로드 → 파일 정상 저장 확인
```

### 시나리오 B: 엣지 케이스 검증
```
1. 크롤링 차단 사이트 (403) → 에러 메시지 표시 확인
2. 잘못된 URL 입력 → 유효성 에러 메시지 확인
3. 응답 느린 사이트 → 타임아웃 처리 확인
```
